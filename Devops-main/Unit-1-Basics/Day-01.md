## 🧠 What is DevOps?

DevOps is a combination of:

* **Development (Dev)** → Writing code
* **Operations (Ops)** → Deploying and maintaining applications

👉 DevOps connects both teams to work together efficiently.

---

## 🔴 Traditional Software Development

Earlier:

* Developers and Operations worked separately
* This caused:

  * Delays in deployment
  * Communication gaps
  * More errors in production

---

## 🟢 DevOps Approach

DevOps solves this by:

* Improving collaboration
* Automating processes
* Making development faster and smoother

---

## 🎯 Why DevOps is Needed

* Faster software delivery
* Better communication
* Fewer errors
* Quick updates and bug fixes
* Automation of repetitive tasks

---

## 📦 Basics of Containers

Containers are lightweight environments used to run applications.

---

### 🕰️ How Technology Evolved

1. **Physical Systems**

   * One system → one application
   * Wastage of resources

2. **Virtual Machines**

   * Multiple systems on one machine
   * But heavy and slow

3. **Containers**

   * Lightweight
   * Fast
   * Efficient

---

### ⚡ Features of Containers

* Share host operating system
* Require less memory
* Start quickly
* Provide isolation between applications

---

### 🏠 Real-Life Example

* 🏡 Independent House → Single system
* 🏢 Apartment → Virtual Machines
* 🛏️ PG System → Containers

👉 Containers share resources but still keep applications separate.

---

## 🖥️ Virtualization Basics

Virtualization means running multiple systems on a single physical machine.

---

## ⚙️ What is a Hypervisor?

A **hypervisor** is software that:

* Creates virtual machines
* Manages system resources
* Controls how VMs run

---

### 🏢 Simple Example

* Building → Physical machine
* Flats → Virtual Machines
* Manager → Hypervisor

---

## 📉 Limitation of Virtual Machines

* Require full operating system
* Consume more memory
* Slower compared to containers
